CR Implementation report - longdesc

This document shows that the longdesc attribute as described in the HTML Image Description Extension (longdesc) specification is implementable, and that independent implementations are highly interoperable. In summary, there are a number of independent implementations that demonstrate interoperability in each of the specification's requirements, for a variety of real-world scenarios .

The specification explicitly defines 3 requirements for User Agents in section 3.03, plus a requirement on how the attribute is treated in the DOM in section 3.1.1, which in practice applies to User Agents:

  1. If the longdesc value is valid, User agents must make the link available to all users through the regular user interface(s).
  2. If the longdesc value is valid, User agents must expose the link to relevant APIs, especially accessibility-oriented APIs.
  3. User agents should enable users to discover when images in a page contain a long description.
  4. The longdesc IDL attribute must reflect the HTML content attribute longdesc.

"Plan-2014" recognises that a User Agent may be a single piece of software such as a browser, but may in practice consist of a set of software that a user runs togther, such as a browser plus extensions, assistive technologies, and OS-based "helpers" or specific settings.

A set of tests are used to test the following requirements for User Agents (in section 3.03):

  1. If the longdesc value is valid, User agents must make the link available to all users through the regular user interface(s).
  2. User agents should enable users to discover when images in a page contain a long description.

The tests cover a variety of different scenarios, for example whether the longdesc refers to a description on the same page as or a different page from the image, where there is a base element used on the page, where a data: URI is used as the value of the longdesc, etc. The results of these tests are shown in table 1, but in summary there are a number of User Agents (or composite user agents) in public release that enable discovery of and access to descriptions of images linked by a longdesc attribute.

The same set of tests were used to demonstrate that user agents make the relevant information available through at least the MSAA accessibility interface, as shown in Table 2, as per the requirement

  1. If the longdesc value is valid, User agents must expose the link to relevant APIs, especially accessibility-oriented APIs.

In addition, since the DOM is used as an accessibility API by many systems (e.g. screenreaders), the test showing that longdesc is reflected in the DOM is evidence that this requirement is met by multiple user agents.

The test that longdesc is reflected in the DOM primarily shows that the requirement

  • The longdesc IDL attribute must reflect the HTML content attribute longdesc.

(section 3.1.1) is met by multiple browsers, as per Table 3.
